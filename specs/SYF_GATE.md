# SYF Gate

## Definition

**SYF Gate** is a law-layer security primitive whose sole function is to
separate **cryptographic identity** from **capacity to act**.

The Gate does not authorize actors.
It evaluates **whether an action can exist** under strict systemic invariants.

SYF Gate operates in **fail-closed mode**, is **fully deterministic**, and
contains **no governance, no oracle, no discretion, and no interpretation of intent**.

> Signature ≠ Authorization  
> Identity ≠ Capacity  
> Intelligence ≠ Permission

---

## Scope

SYF Gate:
- Evaluates a **canonized action request** against invariants
- Produces a **single, final, deterministic verdict**
- Does **not** modify SYF / SyFF laws
- Stores **no intent, no narrative, no history**

Out of scope:
- Governance or administrative control
- Human policies or discretion
- Optimization, scoring, reputation
- Moral alignment
- Prediction, simulation, or intent inference

---

## Canonical Position

SYF Gate is a **law-facing barrier**.

If an action passes the Gate, it is **not approved** —
it is merely **not impossible**.

If an action is blocked, it is **not punished** —
it is **structurally disallowed**.

## Status

This document defines the **sealed canonical scope** of SYF Gate.

Any implementation deviating from this scope
**must not be represented as SYF Gate**.
